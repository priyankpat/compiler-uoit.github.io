Definitions

2.1

Why automata?

  • RE are not immediately algorithmic.

    Given a RE (a?(bc)+)*d, how do we perform matching / searching / capturing on an arbitrary string?

  • Automata to the rescue.

    Automata are symbol state transition systems. They are easy to encode as data structures, and easy to simulate using software.

  • Automata has the right expressiveness to capture regular languages.

2.2

Nondeterministic finite automata (NFA)

Definition:

  • A set of states: $S$
  • An initial state: $s_0$
  • A set of accepting states: $F\subseteq S$
  • An alphabet: $\Sigma$
  • An transition relation: $\Delta: S\times(\Sigma\cup\{\epsilon\})\to\mathrm{Powerset}(S)$

An automata is made up by all these things:

$$A = \left<S, \Sigma, \Delta, s_0, F\right>$$

2.3

Deterministic finite automata (DFA)

A deterministic finite automaton (DFA) is a special type of NFA:

  1. There can be no $\epsilon$ transitions
  2. For each state $s\in S$ and a symbol $a\in \Sigma$, $|\Delta(s,a)|\leq 1$.

    This is the deterministic nature of DFA. Each state is taken to at most one unique next state by a symbol $a$.

3

String acceptance and simulation

3.1

Acceptance by NFA

Let $N$ be NFA, and $x\in\Sigma^*$ be a string. The automaton makes state transitions, making one move for each symbol.

The string $x$ is accepted by $N$ if it is possible to transit from the initial state to any one of the final states by symbols in $x$.

The langauge defined by a NFA, $N$, written $L(N)$ is all the strings that are accepted by $N$.

3.2

Acceptance by DFA

Let $A$ be a DFA, and $x\in\Sigma^*$. There exists at most one unique transition path from the initial state following the symbols of $x$.

We say that $x$ is accepted by $A$ if there exists a path from the initial state to a final state of $A$ following the symbols of $x$.
