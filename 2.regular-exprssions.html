Regular Expressions

1

Syntax of Regular Expressions

  • a, b, c, …

    Single symbols represent constant languages. $L(a) = \{a\}$

  • $e_1 | e_2$

    $L(e_1 | e_2) = L(e_1) \cup L(e_2)$

  • $e_1 e_2$

    $L(e_1 e_2) = L(e_1) \cdot L(e_2)$

  • $e*$

    $L(e*) = L(e)^*$

  • The empty string represents $\{\epsilon\}$.

This is considered the core regular expression. There are many extensions done to the syntax to make expressing certain patterns easier.

  • Dot: .

    Matches any single symbol. This is the abbrevation for a|b|c|d|...

  • Caret: ^

    Matches the start of a string.

  • $

    Matches the end of a string.

  • $e? = \epsilon|e$

    Zero or one occurrence of $e$.

  • $e${m} = $e e e \dots e$ ($m$ times )

    Exactly $m$ occurrences of $e$.

  • $e${m,n}

    At least $m$ occurrences, but at most $n$ occurrences of $e$.

  • $e${,n}

    $m$ is defaulted to zero

  • $e${m,}

    $n$ is defaulted = $\infty$

  • Character classes
    • [abc] matches one symbol in the set $\{a, b, c\}$.
    • [a-z] matches one symbol in the range from a to z inclusively.
    • [^abc] matches one symbol that is anything but $\{a, b, c\}$
  • Predefined character classes

    \w: a symbol that can be part of a word: [a-zA-Z0-9_]
    \W: a symbol not part of word: [^\w]
    \d: a digit symbol
    \D: a non-digit symbol
    \s: a whitespace symbol
    \S: a non-whitespace

To learn more, see

  • https://docs.python.org/2/library/re.html
  • https://docs.oracle.com/javase/tutorial/essential/regex/

Programmatic Regular Expressions

2.1

General usage of regular expressions

  • Matching

    We want to check if a given string satisfies a certain pattern. Formally, we are checking: $$s\in L(e)$$

  • Search

    We want to check if a given string contains a certain pattern. Formally, we are checking: $$s\in \Sigma^*\cdot L(e)\cdot\Sigma^*$$

Note: search is a simple case of match.

  • Capture

    We want to extract the specific parts of a given string. Each part is described by a pattern.

2.2

Matching & Searching

Java offers excellent support for regular expressions.

  • java.util.regex
  • Pattern

    A regular expression pattern

  • Matcher

    A matching session against a specific string.

2.3

Capturing with Groups

  • Defining groups

    Any subexpressions of a regular expression can be enclosed by parentheses (...). Any parenthese enclosed subexpression is called a group.

Example:

$e$ = Name: (\w+)\s+(\w*)
  • Groups are indexes by a positive integer
    • $e$ has two groups, indexed by 1, 2.
    • group 0 is special – it refers to the entire regexp $e$.
    • During matching, each group captures the text that matches the subexpressions of that group.
  • java.util.regexp.Matcher provides methods to access the matched text of the groups.
    • int Matcher.groupCount(): counts the number of groups of a matcher.
    • String Matcher.group(int i): gets the matching of the subexpression for group $i$.
String line;
Pattern namePattern = Pattern.compile("Name: (\w+)\s+(\w*)")
BufferedReader reader = new BufferedReader(
                           new FileReader("text.name"));

while((line = reader.readLine) != null) {
  Matcher m = namePattern.matcher(line);
  while(m.find()) {
    String firstName = m.group(1);
    String lastName = m.group(2);
    out.Printf("%s, %s\n", lastName, firstName);
  }
}
